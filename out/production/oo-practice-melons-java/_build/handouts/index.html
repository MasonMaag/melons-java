The Melon Farm

In this exercise, we’ll take a real world set of objects, data, and behaviors and create several classes that could encapsulate these objects in a Java program. We’ll also create one relationship between two classes.

The scenario:

At a nearby melon farm, a large melon harvest is imminent. The farm grows several types of melons. The farm expects to harvest many melons each day.

In this object-based system, we’ll have two objects:

  • MelonType

  • Melon

Review Classes

Open the files Melon.java, MelonType.java, and Main.java in your editor. Read each file carefully.

Making Melon Types

In the file Main.java, inside the main method, instantiate the class MelonType for each of the above melon types.

How to instantiate a MelonType to represent muskmelons

    "musk",
    "Muskmelon",
    1998,
    "green",
    true,
    true
);

Make Melon Types Array

Continuing in the Main class’s main method, initialize an array that will be able to hold all of the melon type instances you just created. Then, add the melon types to your array.

Making Arrays

There are two ways to create an array in Java.

  1. Initialize an array with a size, and then assign each item into the array using the index.

  2. Use the array “shorthand”, which allows you to define the array and it’s elements all on one line.

Either way, you’ll need to ensure to initialize the datatype of your array like so: MelonType[] melonTypes = YOU_FILL_IN_THE_REST.

Melon Behaviors

All melons can be categorized as sellable or not sellable. A melon is able to be sold if both its shape and color ratings are greater than 5, and it didn’t come from field 3, which was found to have poisonous fertilizer planted by a competing melon farm.

Create a method on the Melon class called isSellable that handles this logic. Some notes on this method:

  • This will not be a static method, since it will deal with the instance of a particular melon.

  • You can refer to the instance of a melon within a method on the Melon class by using the keyword this.

  • Recall that for all java methods, you must specify the return type. Would would be a good return type for this method? A boolean!

Sellability Report

Your last task is to write a method for the Main class, getSellabilityReport, which prints out information about each melon that was harvested, and whether or not it is sellable.

This function should take an array of melon instances as an argument, and return nothing. But, the function will print out who harvested each melon, what field it was harvested from, and whether it is sellable or not. In order to print this report, you’ll access several fields, and call the isSellable method on each melon instance.

The output for this function should look like this:

Harvested by Sheila from Field 2 (CAN BE SOLD)
Harvested by Sheila from Field 2 (NOT SELLABLE)
Harvested by Sheila from Field 3 (NOT SELLABLE)
Harvested by Sheila from Field 35 (CAN BE SOLD)
Harvested by Michael from Field 35 (CAN BE SOLD)
Harvested by Michael from Field 35 (NOT SELLABLE)
Harvested by Michael from Field 4 (NOT SELLABLE)
Harvested by Michael from Field 4 (CAN BE SOLD)
Harvested by Michael from Field 3 (NOT SELLABLE)
